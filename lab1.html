Lab 1: Preguntas

Pregunta Respuesta
¿Cuál es la diferencia entre Internet y la World Wide Web? Internet es una red global de computadoras interconectadas que permite la comunicación y el intercambio de datos entre ellas. La World Wide Web, por otro lado, es un servicio que se ejecuta en la parte superior de Internet y permite acceder y visualizar información a través de páginas web utilizando navegadores web.
¿Cuáles son las partes de un URL? Dominio: la dirección del servidor donde se encuentra el recurso. Ruta: la ubicación específica del recurso en el servidor. Parámetros: datos opcionales que se pueden pasar al servidor. Fragmento: una parte específica del recurso al que se desea acceder.
¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?. GET: solicita datos del servidor. HEAD: solicita encabezados del recurso sin obtener el cuerpo del mensaje. POST: envía datos al servidor para ser procesados. PUT: envía datos al servidor para que sean almacenados en una ubicación específica. PATCH: modifica parcialmente un recurso.DELETE: elimina un recurso del servidor.
¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?. -Se debe utilizar el método POST. Esto se debe a que el método POST envía los datos del formulario en el cuerpo de la solicitud HTTP, lo que es más seguro y adecuado para el envío de datos sensibles como contraseñas
¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL? Normalmente se utiliza el método GET al acceder a una página a través de un URL. Este método solicita y recupera recursos del servidor.
¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué? Sí, es responsabilidad del desarrollador corregir un error 404, ya que indica que el recurso solicitado no se encuentra en el servidor.Esto puede afectar negativamente la experiencia del usuario y la funcionalidad del sitio.
¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué? -Sí, es responsabilidad del desarrollador corregir un error 500, ya que indica un error interno del servidor que impide que se procese la solicitud correctamente
¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados. Un atributo HTML5 depreciado significa que se desaconseja su uso y puede ser eliminado en futuras versiones. Algunos elementos de HTML 4 desaprobados en HTML5 incluyen font, center, strike, entre otros.
¿Cuáles son las diferencias principales entre HTML 4 y HTML5? HTML5 introduce nuevas etiquetas semánticas como header, footer, nav, etc. Soporte mejorado para audio y video. Mayor capacidad para almacenar datos localmente. Nuevas API para aplicaciones web más interactivas.
¿Qué componentes de estructura y estilo tiene una tabla? Estructura: table, tr (fila), td (celda), th (encabezado de celda). Estilo: se pueden aplicar estilos CSS para modificar el aspecto de la tabla, como el color de fondo, el tamaño de la fuente, el borde, etc
¿Cuáles son los principales controles de una forma HTML5? input: para ingresar datos. select: para seleccionar una opción de una lista desplegable. textarea: para ingresar texto multilínea. button: para enviar el formulario o realizar alguna acción específica. checkbox y radio: para seleccionar opciones múltiples o únicas, respectivamente.
