Diferencias y similitudes entre Java y JavaScript:

Diferencias:

  • Java es un lenguaje de programación orientado a objetos de propósito general, mientras que JavaScript es un lenguaje de secuencias de comandos principalmente utilizado para programación web.
  • Java requiere un entorno de ejecución (JVM) para ejecutar código, mientras que JavaScript se ejecuta en un navegador web o en el entorno Node.js.
  • Java utiliza tipado estático, lo que significa que los tipos de datos de las variables se declaran explícitamente, mientras que JavaScript es de tipado dinámico.
  • Java se compila en código de bytes antes de la ejecución, mientras que JavaScript se interpreta en tiempo de ejecución.
  • Java tiene fuertes restricciones de seguridad debido a su entorno de ejecución controlado, mientras que JavaScript ejecuta en un sandbox dentro del navegador, limitando su acceso al sistema operativo.

Similitudes:

  • Ambos lenguajes utilizan una sintaxis similar a C.
  • Tienen estructuras de control de flujo similares, como bucles y condicionales.
  • Comparten algunas características orientadas a objetos, como clases, métodos y herencia.

Métodos del objeto Date:

  • getDate(): Obtiene el día del mes (del 1 al 31).
  • getMonth(): Obtiene el mes (del 0 al 11).
  • getFullYear(): Obtiene el año.
  • getHours(): Obtiene la hora (0-23).
  • getTime(): Obtiene el tiempo en milisegundos desde el 1 de enero de 1970.

Métodos de los arreglos en JavaScript:

  • push(): Añade uno o más elementos al final del arreglo y devuelve la nueva longitud del arreglo.
  • pop(): Elimina el último elemento de un arreglo y devuelve ese elemento.
  • splice(): Cambia el contenido de un arreglo eliminando o reemplazando elementos existentes y/o agregando nuevos elementos.
  • slice(): Devuelve una copia superficial de una porción del arreglo en un nuevo arreglo.
  • indexOf(): Devuelve el primer índice en el que se encuentra un elemento dado en el arreglo, o -1 si no está presente.

Declaración de una variable con alcance local dentro de una función:

En JavaScript, puedes declarar una variable con alcance local dentro de una función utilizando la palabra clave var, let, o const, dependiendo del comportamiento que desees:

function miFuncion() {
    var variableLocal = 10; // Declaración con var
    let otraVariable = 'Hola'; // Declaración con let
    const PI = 3.1416; // Declaración con const
}

Implicaciones de utilizar variables globales dentro de funciones:

El uso de variables globales dentro de funciones puede tener varias implicaciones negativas:

  • Contaminación del espacio global: Puede conducir a la colisión de nombres de variables y funciones, lo que resulta en comportamientos inesperados y difíciles de depurar.
  • Dificultad para rastrear y entender el código: Las variables globales pueden ser modificadas desde cualquier parte del código, lo que hace que sea más difícil rastrear quién las modificó y en qué contexto.
  • Mayor riesgo de errores: Las variables globales pueden ser modificadas por cualquier función en el código, lo que aumenta el riesgo de errores y hace que el código sea menos modular y más difícil de mantener.
  • Problemas de seguridad: Las variables globales pueden ser accedidas y modificadas desde cualquier parte del código, lo que puede plantear problemas de seguridad si contienen datos sensibles o críticos para la aplicación.

Por lo tanto, es una buena práctica minimizar el uso de variables globales y en su lugar, limitar su alcance dentro de funciones o módulos específicos según sea necesario.
